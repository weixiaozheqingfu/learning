Rabbitmq-admin项目主要是为了管理交换器和队列的，
希望在项目启动的时候执行一次交换器和队列的声明就关掉。

但是spring只有消费者监听队列时，或者发送者发送消息时才会触发交换器和队列的创建。


这个项目如果是消费者客户端可以声明队列和交换器的话，还是很好的，可以直接借鉴这个项目的声明创建交换器和队列的方式。

但我们的最优方案是交换器和队列是由管理员统一创建的，管理员不应该消费队列，只应该创建队列和交换器，
所以代码中不能由监听队列的代码，而没有监听队列的代码，spring就不触发队列和交换器的创建。
或许支持，但是我没有发现。
或许我们可以想办法比如也写监听器。但这有风险，因为万一队列原来存在，那么运行该项目就会消费到消息了，而这个项目是不具备消费消息的业务代码的，所以也不行。
该项目必须具备等幂执行特性，可以任意多次声明队列和交换器。


所以这个项目，我们留着，留着如果项目允许消费者客户端创建队列和交换器时，直接拷贝该项目中代码即可。


所以我们会写一个心的admin1项目，该项目使用最原始的mq的api来操作队列和交换器的创建。
这种方式还有一个好处，就是创建完成后，我们可以马上方便的关掉channel和connection。
